## E13 ‚Äî D-class case mining: results

**Method executed** (read-only throughout ‚Äî `git show`/`git archive` into scratch only, zero writes to `lab/swebench_repos`):

1. **Stage 1 (screening, all 300 instances):** for every gold file bgrep *did* return, fetched the file at `base_commit` via `git show`, parsed it with Python's `ast` module to get function/method spans, and classified every uncovered gold line as **D** (its enclosing function ‚âÝ the enclosing function of any returned span, including "no enclosing function" = module-level/import-block) vs **close-miss** (uncovered but same function as a returned span ‚Äî boundary/partial-region miss). 167/300 instances have D-mass > 0, totaling 1989 D-lines (only 11 close-miss lines total ‚Äî confirms D is almost the entire non-coverage story once a file is in-hand, matching the E7 framing). Took the top 30 by per-instance D-mass (816 of the 1989 D-lines, 41%); min D-mass in the set = 19 lines. All 30 turned out to be single-dominant-function cases (one gold function/site accounts for 100% of that instance's D-mass) ‚Äî clean 1:1 gold-vs-chosen comparisons, no averaging artifacts.
2. **Stage 2 (feature battery, top 30):** materialized each repo at `base_commit` via `git archive | tar -x` into a private scratch dir (never touches the shared clone), built the real `lanes2.Corpus` (frozen v7 pipeline) for true corpus-wide idf, recomputed the *exact* pack_regions objective ‚Äî `(idf-weighted term overlap + 0.5¬∑hit-lines) / tiktoken(cl100k) tokens` ‚Äî for every AST function in the gold file, ranked them, and pulled name/call-graph/exception/adjacency/length/rare-term features for gold vs. the top-scoring "chosen" (actually-returned) function in the same file.

Of the 30, **4 (13%) turned out to be import-block/module-preamble misses**, not function-vs-function contests (gold has no enclosing function at all ‚Äî see below). Feature stats below are over the remaining **26 genuine function-vs-function cases**.

**Bottom line on "is a new signal needed":** yes. Median gold rank is 22.5/~50 ‚Äî this is not a near-miss population that a small top-k bump fixes; the density objective itself needs restructuring for length.

### The import-preamble sub-pattern (found, not asked for, but load-bearing)

4/30 top-D-mass cases (`sphinx-doc__sphinx-11445`, `sympy__sympy-12419`, `django__django-11019`, `django__django-11564`) have gold lines with **no enclosing function at all** ‚Äî they're import blocks / module preambles that the gold patch touches (adding an import, a new module-level constant) as part of a larger fix. Checking this pattern across the full 167-instance D population: **26/167 (16%) of D-mass cases have a module-level dominant owner, accounting for 13.3% of all D-mass (265/1989 lines)** ‚Äî almost always concentrated in the first ~5‚Äì30 lines of the file. This is structurally *unfixable* by any function-ranking signal (there's no function to rank) but trivially fixable: force-include a returned file's own import/preamble block at near-zero token cost, independent of term density.

### Five vignettes

**1. `pallets__flask-5063`** (add subdomain info to `flask routes`). Gold: `routes_command` (47 lines, rank 13/34, density 0.0625). Chosen: `_env_file_callback` (21 lines, rank 1, density 0.129 ‚Äî 2.1√ó). The query's generic words "flask"/"routes"/"default" scattered thinly across the real 47-line command collide with `_env_file_callback`'s tight little cluster of "flask"/"env"/"file"/"dotenv". Length ratio 2.2√ó ‚Äî a comparatively *mild* case of the pattern, useful as a baseline.

**2. `django__django-13265`** (`AlterOrderWithRespectTo()` + `Index()` crash). Gold: `MigrationAutodetector.generate_created_models` (174 lines, rank 24/42, density 0.0906). Chosen: `generate_added_indexes`, **same class**, 10 lines, density 0.352 (3.9√ó). Here call-graph actually favors gold cleanly (12 query-matching calls vs. chosen's 2 ‚Äî `AddIndex`/`CreateModel`/`AlterOrderWithRespectTo` etc. are all named almost verbatim in the query) ‚Äî but it loses anyway because the length/density penalty (17.4√ó) is bigger than the call-graph edge. Demonstrates the two signals aren't independent-and-additive in isolation; length dominates.

**3. `sympy__sympy-20639`** (`pi**(1/E)` renders wrong in pretty-printing). Gold: `PrettyPrinter._print_nth_root`, the actual 44-line rendering routine, rank **197 of 225** functions in `pretty.py`, density 0.0306. Chosen (rank 1, density 0.333 ‚Äî **10.9√ó higher**): `PrettyPrinter.emptyPrinter`, a **two-line stub** (`def emptyPrinter(self, expr): return prettyForm(str(expr))`) that happens to contain both "pretty" and "print" ‚Äî both near-universal in a class where every one of ~200 sibling methods is named `_print_X`. This is the cleanest possible demonstration of the objective's failure mode: a trivial boilerplate stub outranks the real 44-line fix purely because the token-count denominator crushes it to near-zero cost.

**4. `sphinx-doc__sphinx-11445`** (`rst_prolog` breaks headings with domain directives). Gold lines are the file's own **import block** (lines 10‚Äì31) plus two small non-function code blocks (80‚Äì86, 91‚Äì96 ‚Äî a `for`/`if` loop body directly under a function, split oddly by the AST attribution). No function-ranking feature applies here at all ‚Äî it's the import-preamble pattern above.

**5. `pytest-dev__pytest-7490`** (dynamically-added `xfail` marker no longer honored). Gold: `pytest_runtest_setup` (13 lines, rank **3/10**, density 0.0906). Chosen: `pytest_addoption` (17 lines, rank 1, density 0.0912 ‚Äî essentially **tied, 1.01√ó**). Gold calls `evaluate_xfail_marks`/`evaluate_skip_marks`/`xfail` ‚Äî direct hits against the query's own vocabulary ‚Äî while chosen calls nothing query-relevant. This is the one case in the whole sample where (a) a k=2/3-regions-per-file bump, or (b) a call-graph tiebreak, would cleanly flip the outcome ‚Äî included specifically as the contrasting "this one actually is a near-miss" case against the other four.

### Ranked recommendation (2 signals, by expected D-mass coverage)

**#1 ‚Äî Dampen the token-count length penalty in the packing objective (targets ~85‚Äì96% of function-vs-function D-mass, i.e. the dominant failure mode).** The evidence is unambiguous and enormous in magnitude: gold is longer than chosen in 25/26 cases, >3√ó longer in 22/26, and the win margin (chosen scores 3.6‚Äì5.5√ó higher density) tracks length almost exactly (`sympy-20639`'s 2-line stub vs. 44-line fix, 22√ó length ratio, is the extreme but representative case). Concretely: replace `gain / tok` with something sub-linear in `tok` (e.g. `gain / tok**0.7` or `gain / (tok + const)` with a larger additive floor, or BM25-style `b`-parameter length saturation applied to the *region* rather than the *file*) so a real function with modest-but-genuine density isn't crushed relative to a short function that got lucky on 1‚Äì2 common tokens. This must be swept and gated exactly the way `w_name` was (lab/region_eval_lanes.py's existing harness, full-300 hunk_line_recall, not just this 26-case sample) before shipping ‚Äî the same class of change (a new multiplicative/normalization knob in `pack_regions`) has burned the project once already (issue #4/#8's `w_name` regression), so treat this as a hypothesis to gate, not a guaranteed win.

**#2 ‚Äî Same-class sibling-method locality boost (targets ~42% of cases, smaller and likely overlapping with #1, but structurally safer).** 11/26 cases already have the *correct class* anchored via some other method's density win (`django-13265`, `sympy-20639`, `sympy-16503`, `sympy-23191`, etc.) ‚Äî the file/class localization succeeded, only the specific-method greedy pick failed. A bounded fix in the same spirit as the existing anchor-symbol force-include mechanism (`anchor_def_symbols`/`pack_regions`'s forced pass-1 pick): once a class is anchored by any of its methods scoring above some threshold, give sibling methods of that *same, already-selected* class a marginal-gain bonus in pass-2's greedy loop. This has a much smaller blast radius than symbol-name weighting (it can only ever expand depth *within an already-committed class*, never pull in new unrelated files), which is presumably why it's more likely to avoid `w_name`'s fate ‚Äî but it too needs the same full-300 gate before being trusted.

**Explicitly deprioritized:** symbol-name matching (feature b) ‚Äî only 23% clean discrimination here, and independently measured to regress the shipped engine when tried at any positive weight. Exception-type matching (feature d) ‚Äî narrow applicability (31%) and weak even then. **Bonus, near-free, orthogonal fix:** force-include each returned file's own import/preamble block (first ~20‚Äì30 lines) regardless of term density ‚Äî costs almost nothing token-wise and covers ~13% of D-mass that no ranking signal can ever reach.

**Honest null:** feature (b) name-match is close to a null result here (23% clean discrimination, and both gold and chosen have query-term name overlap equally often at 69% ‚Äî the signal is noisy, not absent, but it's the weakest of the seven and already known to hurt in aggregate).
